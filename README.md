# Общий город

Кооперативная градостроилка в реальном времени: общая казна на всех, налоги,
жители со своими потребностями по пирамиде Маслоу, живые районы и ежедневная
газета. Node + Express + WebSocket, запуск и туннель — как у gnis.

## Запуск

```bash
npm install
npm start
```

`http://localhost:3000` → имя → «Основать город» → код раздаёшь остальным.

## Игроки с других устройств (туннель)

```bash
cloudflared tunnel --url http://localhost:3000
```

Сервер и туннель — два процесса. Рвёт домашняя сеть — раздать интернет с
телефона. В общей Wi-Fi работает и по локальному IP без туннеля.

## Тест вдвоём на одном компе

Личность игрока в `sessionStorage` (изолирован по вкладке): две вкладки = два
игрока, инкогнито не нужно.

## Экономика: общая казна, налоги, содержание

Казна одна на весь город. Здания строятся и сносятся из неё (снос возвращает
50%), налоговую политику может менять любой игрок.

**Доход — налоги** (три ползунка, 0–40%):

| Налог | С чего | По счастью бьёт |
|-------|--------|-----------------|
| Жилой | ценность домохозяйств (население × уровень) | сильно |
| Коммерческий | оборот магазинов/кафе/спортзалов/театров | мягко |
| Имущественный | плоско за каждое здание | почти нет |

**Расход — содержание**: каждое здание съедает `upkeep` за тик (дороги/дома — 0,
парк — 1, торговля — 3, школа — 4, клиника/полиция/театр — 5). Спамить нельзя.
Итог за тик = налоги − содержание. Уйдёшь в минус — казна обнуляется, включается
**дефицит**: счастье падает по всему городу, стройка блокируется.

Смысл: чем больше населения на одну услугу-кластер, тем прибыльнее; ставка —
выбор «доить сейчас или растить город».

## Ресурсы: еда и вода (общий склад)

Первая ступень Маслоу — это уже не «колодец рядом = галочка», а живой баланс.
Колодцы и фермы **производят** воду и еду в общий городской склад, жители её
**потребляют**, склад копит излишки как буфер.

- Колодец качает воду, только если **в его радиусе есть тайлы воды** (ставь у
  озера/реки); ферма родит еду пропорционально **траве в радиусе**.
- Каждый житель ест 1 еду и 1 воду за тик; чем больше население, тем больше нужно
  производителей.
- Потребность «Вода/Еда» у дома закрыта, если на **общем складе хватает ресурса
  на всех** — радиус производителя на покрытие домов больше не влияет,
  распределение идёт через склад. Кончился ресурс — потребность гаснет у всех,
  уровень домов падает.

Всё видно в **верхней строке активов**: 💰 казна, 💧 вода, 🍞 еда, 🪵 дерево,
🪨 камень (со стрелкой прихода/расхода), 👥 население, 💼 занятые/места, ✨ вера.
Ресурс в дефиците подсвечивается красным. Вера пока = население × коэффициент
(задел: центр города и радиус застройки от веры — следующим куском).

## Интерфейс

Газета «Городские вести» — отдельным полем под игровым, всегда на виду. При
наведении на ресурс в верхней панели показывается разбивка: сколько приходит
(и откуда) и сколько тратится за тик и за день. Курсоры других игроков видны на
поле в их цвете. Районы можно переименовать (список «Районы» на панели, кнопка
✏️). При наведении на школу подсвечиваются дома: зелёным — уже образованные,
синим — обучающиеся сейчас, красным — в зоне, но без места (лимит исчерпан).

## Образование, знания и вера

Школа образует жителей в своей зоне (до вместимости, по таймеру в ~3 дня; без
школы образованность тает). Образованные жители дают **право на апгрейд**: дом
улучшается только если его жители образованы, а производство — только если в
районе достаточно образованных работников. На образованных домах значок 🎓.

Но образование не бесплатно: суммарные **знания** города (🎓 в панели)
**понижают веру** (✨). Вера растёт от населения и падает от знаний; чем меньше
вера — тем меньше радиус застройки (базовый радиус при этом неприкосновенен).
Чтобы вернуть территорию, строй **церкви** ⛪ — они окормляют жителей в своей
зоне и поднимают веру, компенсируя просвещение. Так возникает выбор: умный город
богаче и растёт в уровнях, но требует веры, иначе съёживается.

## Уровни зданий

Любое здание (кроме центра и дороги) улучшается инструментом «⬆️ Улучшить» до
3-го уровня за деньги и ресурсы. Уровень показан золотыми точками на здании. С
уровнем растут: добыча производителей, радиус и оборот сервисов, рабочие места,
вывод завода, а у домов — вместимость (5 → 8 → 11). Стоимость улучшения растёт с
уровнем. Уровень 3 требует образования: дом — свою образованность ≥60%, производство (включая завод) — ≥40% образованных в районе. Уровни 1→2 доступны за ресурсы. Требование показано в подсказке.

## Дороги и подключение

Дороги — это **слой поверх тайлов**, а не здание в сетке: их «красят» дешёвым
инструментом (5$), на дорогу нельзя поставить здание, а под здание дорогу — тоже
(сначала снеси). Дорога кладётся только внутри города; на лесу — вырубает его.

Главное правило: **здание работает, только если рядом (по стороне) есть дорога.**
Отключённое здание не даёт покрытия/добычи/рабочих мест/оборота и не вредит
соседям; отрезанный от дороги дом получает −25 к счастью, не растёт и пустеет.
На поле такие здания показаны полупрозрачными с красной меткой ⚑, а в подсказке
пишется «нет дороги рядом — не работает». Дорога рядом с добытчиком/сервисом также
даёт прежний бонус +1 к радиусу.

## Центр города и вера

Партия начинается с **выбора места под центр** (🚩): первый игрок кликает по
свободной клетке, пока флаг не поставлен — строить нельзя (палитра серая, кроме
флага). От центра идёт **радиус застройки** — строить можно только внутри него,
снаружи всё затемнено и обведено пунктиром.

Радиус растёт с **верой**: `радиус = 3 + ⌊вера × 0.15⌋`, где вера = население × 2.
Пустой город начинает с радиуса 3; чем больше жителей — тем шире можно строить.
Пока вера завязана только на население; позже здания будут её давать или отнимать
(тогда же продумаем аккуратную потерю домов при сжатии — сейчас за пределами
радиуса просто нельзя строить новое, existing не трогаем). Центр снести нельзя.

Крутилки: `FAITH_BASE_RADIUS`, `FAITH_RADIUS_COEF`, `FAITH_PER_POP`.

## Строительные ресурсы: дерево и камень

Здания стоят не только денег, но и ресурсов. Их добывают:

- 🪚 **Лесопилка** — дерево из **леса** в радиусе (нет леса рядом — простаивает).
- ⛏ **Каменоломня** — камень из **скал** в радиусе (ставится на траве у скал).

Дерево и камень копятся в общем складе — жители их не едят, тратятся только при
стройке. Город стартует с запасом (`START_WOOD`/`START_STONE`, по умолчанию
30/20), чтобы поставить первые здания и саму добычу (лесопилка — только деньги).
Стоимость в ресурсах видна в палитре (`$50 🪵5 🪨3`), не хватает — здание серое.
Снос возвращает 50% денег и ресурсов.

## Размещение по ландшафту и истощение тайлов

Каждое здание знает, где его можно ставить (поле `allow` в каталоге, по умолчанию
трава+лес). Ферма — только на траве, вода и скалы закрыты для всех. Стройка на
лесу **вырубает** его — тайл становится травой (и рядом с ним лесопилки теряют
сырьё: честный размен «застройка ↔ добыча»).

Лес, скалы и вода **истощаются**: у каждого такого тайла запас (`TILE_RESERVE`,
по умолчанию 100). Лесопилка/каменоломня/колодец тянут ресурс из тайлов в своём
радиусе, тайл беднеет (на карте **бледнеет к траве**) и, опустев, превращается в
траву — добыча там прекращается, приходится тянуться к новым залежам. Наведи на
тайл — в подсказке виден остаток (`запас 42/100`). Высаживание леса/восстановление
— задел на будущее.

## Рабочие места

Жители не берутся из воздуха. Дом при постройке заселяет одна семья, но дальше
люди приезжают только если есть счастье, место в доме **и свободная работа**.

Рабочие места дают не-жилые здания: завод — 8, клиника/полиция/школа — по 4,
магазин/кафе/спортзал/театр — по 3, колодец/ферма — по 1, парк и дороги — 0.

- Налог платят только **занятые** — безработные жилого налога не приносят.
- Безработица (жителей больше, чем мест) режет счастье всему городу → отток.
- Нет свободных мест — новые жители не едут.

Перекос в дома = безработица и отток; перекос в рабочие места = никто не едет.
Строку `💼 занятые/места` видно в бюджете.

## Ландшафт

Карта генерится при создании города: трава, лес, вода, скалы. Вода и скалы
**непроходимы** — на них не построить (превью краснеет). Лес и вода рядом дают
дому небольшой бонус к счастью (природа). Тайлы можно заменить своими картинками
(см. `public/sprites/README.md`, папка `terrain/`).

## Жители: потребности, счастье, миграция

Потребность закрыта, если рядом (в радиусе) есть подходящий провайдер
(соседство с дорогой +1 к радиусу). Уровень счастья дома = сколько ступеней
пирамиды закрыто **подряд снизу**.

| Уровень | Потребность | Чем закрыть |
|---------|-------------|-------------|
| **1** Физиология | Вода / Еда | 💧 Колодец / 🌾 Ферма |
| **2** Безопасность | Здоровье / Порядок | 🏥 Клиника / 🚓 Полиция |
| **3** Принадлежность | Сообщество | 🌳 Парк / ☕ Кафе |
| **4** Уважение | Признание | 🏪 Магазин / 🏋 Спортзал |
| **5** Самореализация | Развитие | 🏫 Школа / 🎭 Театр |

**Счастье (0–100)** складывается из уровня Маслоу, тесноты, налогов, событий
района, вредных зон соседей, безработицы и природы рядом (лес/вода).

**Формы зоны покрытия** у зданий разные: колодец — ромб, ферма/магазин —
квадрат, клиника/полиция/парк/театр — круг, школа — крест (вдоль улиц). Размер
видно в подсказке палитры, при установке (превью под курсором на пустой клетке)
и по наведению на готовое здание — зелёным польза, красным вред. Дорога рядом
даёт +1 к радиусу, пробки — снимают.

**Вредные воздействия** — некоторые здания портят показатели соседям:

| Здание | Вред в зоне | Плюс |
|--------|-------------|------|
| 🏭 Завод | загрязнение: снимает «Здоровье», −20 к счастью (круг r3) | большой коммерческий оборот в казну |
| 🎭 Театр | шум: −10 к счастью (круг r2) | закрывает «Развитие» |
| ☕ Кафе | шум: −6 к счастью (ромб r2) | закрывает «Сообщество» |
| 🏋 Спортзал | шум: −5 к счастью (квадрат r1) | закрывает «Признание» |

**Население (раз в день):** дома строятся **пустыми** и заселяются притоком.
- дом привлекателен (счастье ≥40) и есть место и свободная работа → приезжают
  жители; чем счастливее дом, тем быстрее (счастливые — по 2/день). Хватает
  закрытых базовых нужд (вода/еда/здоровье/порядок), чтобы квартал потихоньку рос;
- несчастны (≤30) → уезжают. Накопили на переезд → перебираются в район получше;
  денег нет → покидают город.

Дом хранит население/вместимость и накопления. Наведи курсор — увидишь
население, счастье, уровень, накопления, вредные зоны рядом и разбор
потребностей по галочкам. На плитке: `жители/вместимость` и точка счастья.

## Живой город: районы, дни и газета

Город начинается **единым посёлком**. По мере роста населения появляются
**районы** (Вороной): при первом пороге город делится надвое, дальше самый
крупный район раскалывается — до потолка `DISTRICT_COUNT`. Границы двигаются
вслед за застройкой. Раз в `DAY_TICKS` тиков — новый день и **газета**. События:

| Событие | Когда | Эффект |
|---------|-------|--------|
| 💅 Зазнайство | район сильно богаче остальных | не пускает чужаков в свои магазины/кафе/парки |
| 🚨 Криминал | копится напряжение (нет полиции) | падает безопасность (без клиники — и здоровье), границы дёргаются |
| 🦠 Эпидемия | переполненный район (клиника снижает шанс) | карантин: уровень домов → 0 |
| 🎉 Фестиваль | театр + парк/кафе в районе | закрыта потребность «Сообщество» |
| 🚧 Пробки | ≥4 провайдера и ни одной дороги | радиус обслуживания −1 |
| 📈📉 Бум / Отток, 🗺 Передел | рост/убыль населения, сдвиг границ | заголовок |
| 🏙 Новый район | город дорос до порога населения | территория делится, появляется новый район |

### Добавить своё событие

Функция `(room, ctx, news)` в `server.js` + строка в `DAY_EVENTS`. `ctx` несёт
агрегаты по районам; заголовок — `news.push({ kind, text })`; механический
эффект — модификатор района `d.mods.<имя> = дни`, который читает `computeSim`.

## Свои спрайты вместо эмодзи

Клади картинки в `public/sprites/`, имя = тип здания (`house.png`, `park.png`…).
Сервер сканирует папку при входе игрока — рестарт не нужен. Форматы
png/webp/jpg/gif/svg. Список имён — в `public/sprites/README.md`.

## Конфиг (env)

| Переменная | Что | Дефолт |
|------------|-----|-------:|
| `PORT` | порт | 3000 |
| `GRID_SIZE` | размер поля | 18 |
| `START_TREASURY` | стартовая казна | 1000 |
| `HOUSE_CAP` | вместимость дома | 5 |
| `TICK_MS` | длина тика, мс | 3000 |
| `DAY_TICKS` | тиков в дне | 8 |
| `DISTRICT_COUNT` | потолок числа районов | ~4 |
| `FIRST_DISTRICT_POP` | население для первого деления | 10 |
| `DISTRICT_STEP` | +население на каждый следующий район | 12 |
| `ROOM_TTL_MS` | хранить пустую комнату, мс | 900000 |

```bash
GRID_SIZE=24 START_TREASURY=1500 TICK_MS=2000 npm start
```
